# Bernd's Equity Factor Lab

Interactive research dashboard for exploring 212 equity factors from Chen & Zimmermann (2022), "Open Source Cross-Sectional Asset Pricing." Site name: **Bernd's Equity Factor Lab**. GitHub: `bwuebben/opensource_ap`.

## Project Structure

```
opensource_ap/
├── data_pipeline/           # Python data download & analytics
│   ├── .venv/               # Python virtual environment
│   ├── cache/               # Cached raw downloads (parquet, CSV, enriched metadata)
│   ├── download_data.py     # Downloads factor returns + signal docs from openassetpricing package
│   ├── download_macro.py    # Downloads FRED-MD macro data (needs FRED_API_KEY env var)
│   ├── compute_styles.py    # Computes equal-weighted style group returns (14 styles)
│   ├── compute_walkforward.py  # Walk-forward optimized multi-factor portfolio (max Sharpe / max return)
│   ├── compute_research.py  # Post-pub decay, crisis perf, crowding, factor momentum, tail risk
│   ├── compute_deciles_and_new_analytics.py  # Decile decomp, decades, seasonality, timing, replication
│   ├── custom_fred_tickers.json  # User-defined additional FRED series to fetch
│   ├── README.md            # Pipeline documentation
│   └── DATA_GUIDE.md        # JSON schema guide for building apps with this data
├── dashboard/               # React + Vite + TypeScript frontend
│   ├── public/data/         # 33 JSON files consumed by the frontend (generated by pipeline)
│   └── src/
│       ├── pages/           # 29 page components
│       ├── components/      # Layout, FactorName, FactorInfoModal, FactorContext, Methodology, etc.
│       ├── PlotlyChart.tsx   # Custom Plotly wrapper with onRelayout support
│       ├── dataLoader.ts    # Fetch functions for all JSON data files
│       └── hooks.ts         # useData hook for async data loading
└── refresh_data.sh          # One-command data refresh (runs all pipelines)
```

## Tech Stack

- **Frontend**: React 19, Vite, TypeScript, Tailwind CSS v4
- **Charts**: plotly.js-dist-min (custom wrapper, NOT react-plotly.js due to CJS import issues with Vite)
- **Math**: MathJax v3 via CDN for methodology sections
- **Data Pipeline**: Python 3.13, pandas, numpy, scipy, openassetpricing package
- **Macro Data**: fredapi (FRED API key: set FRED_API_KEY env var)
- **No backend server** - frontend reads static JSON files from public/data/

## Running

```bash
# Start dashboard (data must already exist in dashboard/public/data/)
cd dashboard && npm run dev

# Production build
cd dashboard && npm run build && npm run preview

# Refresh all data (download + compute)
./refresh_data.sh

# Or run pipeline steps individually:
source data_pipeline/.venv/bin/activate
python3 data_pipeline/download_data.py          # factor returns + signal docs
python3 data_pipeline/compute_styles.py          # style group returns
python3 data_pipeline/compute_walkforward.py     # walk-forward optimization (slow ~2min)
python3 data_pipeline/compute_research.py        # research analytics
python3 data_pipeline/compute_deciles_and_new_analytics.py  # decile data + new analytics
FRED_API_KEY=<key> python3 data_pipeline/download_macro.py  # macro data
```

## Data Sources

- **openassetpricing.com** - Chen & Zimmermann (2022), Critical Finance Review
- 212 factors, monthly long-short returns (percentage form, divided by 100 in pipeline)
- Signal documentation with authors, year, journal, definitions, quality ratings
- Decile portfolio returns (EW) for long/short leg decomposition
- No WRDS access needed for portfolio returns; stock-level signals would require WRDS

## Macro Data

- **Primary**: FRED-MD (McCracken & Ng) CSV — must be manually downloaded from https://www.stlouisfed.org/research/economists/mccracken/fred-databases and placed in `data_pipeline/cache/` (auto-download blocked by 403). Pipeline auto-detects CSV files with "md", "current", or "fred" in the filename.
- **Supplemental**: 5 FRED API series (USREC, NFCI, ANFCI, CFNAI, USEPUINDXM)
- **Custom**: Additional tickers via `data_pipeline/custom_fred_tickers.json` (currently: CPILFESL)
- **Enrichment**: Series descriptions cached in `data_pipeline/cache/fred_md_enriched_meta.json`
- **Total**: 132 macro series in output

## Deployment

- **Live site**: https://bwuebben.github.io/opensource_ap/
- **Hosting**: GitHub Pages, auto-deploys via GitHub Actions on push to `main`
- **Workflow**: `.github/workflows/deploy.yml` — builds with `npm run build` in `dashboard/`, uploads `dist/` to Pages
- **Base path**: Vite `base: '/opensource_ap/'` — all asset and data URLs use `import.meta.env.BASE_URL`
- **Data fetch paths**: All `fetch()` calls use `dataUrl()` helper from `dataLoader.ts` which prepends the base path. Never hardcode `/data/...` paths.
- **SPA routing**: `BrowserRouter` with `basename={import.meta.env.BASE_URL}`. `public/404.html` handles GitHub Pages deep-link redirects via sessionStorage.
- **Lazy loading**: 26 pages use `React.lazy()` for code splitting. Vite generates separate JS chunks per page.
- **FRED API proxy**: Only works in dev mode (`npm run dev`). The Macro Dictionary custom ticker fetcher won't work on the deployed site due to CORS.

## Key Technical Decisions

- **Returns are in percentage form** from the source (e.g., 1.68 = 1.68%). Pipeline divides by 100 to get decimals before computing stats. This caused early bugs with cumulative returns overflowing to infinity.
- **Plotly import**: react-plotly.js has CJS/ESM interop issues with Vite. Using a custom wrapper component (`PlotlyChart.tsx`) that calls `Plotly.react()` directly via plotly.js-dist-min. Wrapper supports `onRelayout` for y-axis auto-fit.
- **Date alignment**: Factor returns use end-of-month dates (2020-01-31), macro data uses first-of-month (2020-01-01). Factor-Macro regression page aligns on YYYY-MM.
- **FRED-MD date parsing**: CSV uses 2-digit year sasdate format (M/D/YY). Pandas maps years >68 to 19xx, <=68 to 20xx. Pipeline corrects dates beyond 2030 by subtracting 100 years.
- **JSON sanitization**: Python's json.dump writes Infinity/-Infinity which is invalid JSON. All pipeline scripts sanitize inf/nan to null before saving.
- **React hooks rule**: All hooks must be called before any early returns in components (caused crashes in FactorBrowser, Seasonality).
- **MathJax in JSX**: Dollar-sign math notation like `$S_{i,t}$` must be wrapped in `{"..."}` strings to prevent JSX from parsing `{i,t}` as JavaScript expressions.
- **Navigation**: Sidebar with collapsible sections. Reference section at bottom for Data Sources page.
- **Heatmap color scales**: Use percentile clipping (2nd/98th or 5th/95th) to prevent outlier washout. 5-stop gradients for better contrast on dark background.
- **Style factors**: Available across Returns pages, Factor-Macro Regressions, and Correlation page. Prefixed with "Style: " and shown in optgroups at top of dropdowns.
- **Vite proxy**: `/fred-api` proxies to FRED API to bypass CORS for the Macro Dictionary custom ticker fetcher.

## Pages (29 total)

### Overview
- Dashboard with summary cards, Sharpe/t-stat distributions, top 20 chart, category pie

### Factors
- **Factor Browser** - Sortable/filterable table, expandable detail rows
- **Factor Dictionary** - Full signal documentation with definitions, signal type legend, info modal with Scholar + SSRN links
- **Factor Scatter** - Plot any stat vs any stat, colored by economic category
- **Factor Screener** - Multi-criteria filter with presets
- **Rolling Stats** - Rolling Sharpe, vol, mean, pairwise correlation (individual + style factors)
- **What Worked Recently** - 1/3/6/12 month performance leaderboard

### Returns
- **Cumulative Returns** - Multi-factor overlay with range slider, log scale (individual + style factors)
- **Monthly Heatmap** - Year x Month return heatmap with percentile-clipped color scale (individual + style factors)
- **Drawdowns** - Filled drawdown curves with return/DD ratio table (individual + style factors)
- **Correlation** - Top-50 factor correlation heatmap + 14x14 style factor correlation heatmap
- **Annual Returns** - Bar charts with side-by-side comparison (individual + style factors)
- **Seasonality** - Calendar month patterns, cross-factor average, most-seasonal heatmap
- **Decade Performance** - Sharpe ratio heatmap by decade (1920s-2020s)

### Styles & Portfolios
- **Style Analysis** - 14 equal-weighted style groups, cumulative/rolling/annual, correlation matrix
- **Style Portfolio** - Composite portfolio builder, toggle styles, live stats update
- **Walk-Forward** - Expanding-window optimization, annual rebalance, equal-weight benchmark, weight evolution

### Research
- **Replication Tracker** - Original vs replicated t-stats, replication rate (80.5%)
- **Post-Publication Decay** - Pre vs post-pub Sharpe comparison, decay by category/decade
- **Long/Short Decomposition** - Decile D1 vs D10 returns, long-driven vs short-driven
- **Regimes & Crises** - 9 crisis windows, top/bottom performers, cross-crisis heatmap
- **Crowding & Overlap** - Rolling avg pairwise correlation among top 30 factors
- **Factor Momentum** - 4 strategies: quintile L/S, long-only, z-score L/S, z-score long-only
- **Tail Risk** - VaR, CVaR, max DD duration, worst drawdown episodes
- **Factor Timing** - Conditional Sharpe by macro regime terciles

### Macro
- **Macro Dashboard** - 132 FRED series with recession shading, checkbox selector
- **Macro Dictionary** - Searchable reference with inline charts, custom FRED ticker fetcher
- **Factor-Macro Regressions** - OLS regression of any factor/style on any macro variable, sensitivity rankings

### Reference
- **Data Sources** - Data provenance, transformations, missing data treatment, pipeline details
